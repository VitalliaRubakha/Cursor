Основні команди Git:

  • git config --global user.name blabla - дана командає дає встановити ім'я користувача, при завантаженні git на пк
  • git config --global user.email blabal@bla - дана командає дає встановити пошту користувача, при завантаженні git на пк
  • git init - щоб перетворити звичайний каталог на Git репозиторій і почати версіонувати файли в ньому
  • git status - показує стан файлів у робочому каталозі та індексі: які файли змінені, але не додані в індекс; які очікують комміта в індексі.
  • git add - додає вміст робочого каталогу в індекс (промежуточну область) для наступного комітату.
  • git commit - бере всі дані, додані в індекс за git addдопомогою , і зберігає їх сліпок у внутрішній базі даних, а потім зрушує вказівник поточної ветки на цей сліпок.
  • git log - використовується для перегляду історії коммітів, починаючи з найсвіжішого та йдучи до витоків проекту.
  • git checkout - використовується для перемикання гілок та вивантаження їхнього вмісту в робочий каталог.
  • git branch - це свого роду "менеджер гілок". Вона вміє перераховувати ваші гілки, створювати нові, видаляти та перейменовувати їх.
  • git push - використовується для встановлення зв’язку з видаленим репозиторієм, вирахування локальних змін, наявних у ньому, і власне їх передачі у вищевказаний репозиторій.
  • git pull - працює як комбінація команд git fetchі git merge, т. е. Git спочатку збирає зміни із зазначеного видаленого репозиторію, а потім намагається розрізати їх з поточною веткою.
  • git merge - використовується для злиття однієї або кількох гілок у поточну. Потім вона встановлює покажчик поточної гілки на результуючий коміт.
  • git clone - працює як обгортка над деякими іншими командами. Вона створює новий каталог, переходить всередину і виконує git initдля створення порожнього репозиторію, потім вона додає новий віддалений репозиторій
  • gitignore - визначає, які файли та папки не слід ігнорувати у даному вихідному коді. Його можна використовувати на основі кожного шляху, щоб правила застосовувались лише до певних папок, але ви також можете створити глобальний файл GITIGNORE, який застосовується до кожного сховища Git, яке у вас є.

Advanced

  • git revert - повна протилежність git cherry-pick. Вона створює новий коміт, який вносить зміни, протилежні до зазначеного комміту, по суті скасовуючи його.
  • git reset - використовується в основному для внесення змін.
  • git rebase - це «автоматизований» cherry-pick. Він виконує ту ж роботу, але для ланцюжка коммітів, тим самим переносячи гілку на нове місце
  • git cherry-pick - бере зміни, що вносяться одним коммітом, і намагається повторно застосувати їх у вигляді нового комміта в поточній гілці
  • git stash - використовується для тимчасового збереження всіх незафіксованих змін для очищення робочого каталогу без необхідності фіксувати незавершену роботу в поточній гілці.
